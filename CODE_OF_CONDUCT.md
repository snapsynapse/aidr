# Code of Conduct

This project uses a small-project maintainer standard: be direct, civil, and evidence-oriented. Disagreement is welcome when it improves the specification, examples, or tooling.

## Expected behavior

- Keep critique about the work, not the person.
- Preserve dissent when it matters; do not pressure contributors to erase well-grounded objections.
- Assume contributors may use different tools, models, providers, editors, and operating systems.
- Respect maintainer decisions about scope, especially the rule that AIDR is one decision, one file.

## Unacceptable behavior

- Harassment, threats, personal attacks, or sustained disruption.
- Publishing private contact information or secrets.
- Misrepresenting model output, provenance, authorship, or arbitration authority.
- Pressuring a model or human participant to rewrite another participant's position as if it were their own.

## Enforcement

The maintainer may edit, hide, or remove issues, comments, or pull requests that violate this code. Serious or repeated violations may result in blocking from the project.

Report conduct concerns privately to `info@snapsynapse.com`.
